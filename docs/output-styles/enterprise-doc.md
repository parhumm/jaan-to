# Enterprise Doc

> Formal enterprise document formatting with metadata headers and structured sections.

---

## What It Does

Formats skill outputs as formal enterprise documents. Use this style when outputs go to stakeholders, leadership, or cross-functional teams.

---

## Formatting Rules

| Rule | Description |
|------|-------------|
| **YAML metadata** | Title, date, author, status, version at top |
| **Executive summary** | First section after metadata |
| **Numbered sections** | Clear hierarchy with numbered headings |
| **Decision log** | Appended at end when applicable |
| **Status badges** | Draft, Review, Approved, or Deprecated |

---

## When to Use

- PRDs for stakeholder review
- Technical specs for cross-team consumption
- Proposals requiring formal structure
- Documents that need version tracking
